# 简谱 jianpu-ly 格式纠错提示词

> 本文件用于向大语言模型（LLM）说明 `.jianpu.txt` 简谱文件的格式规范，  
> 防止 LLM 在纠错过程中引入格式错误（尤其是小节拍数溢出）。

---

## 为什么需要这份提示词

经实测，LLM 对此格式的常见破坏：

| 错误类型 | 示例（4/4拍）| 错误拍数 |
|---|---|---|
| **将整条节奏重写/简化** | `d1' d0 s0 sb2' s0 qb3' qb3' qb2' q1' s0 q0.`（4.0拍）→ `q1' qb2' qb3' qb3' qb2' q1' 0 -`（**5.0拍**）| 原来正确，改后变错 |
| 在休止符后多加 `-` | `q1' q2' q3' q4' 0 -`（将四分休止改为二分休止）| 5 拍 → 应 4 拍 |
| 删除或添加了某个音符 | 在原 4 拍小节内多插入一个 `q` 音符 | 4.5 拍 → 应 4 拍 |
| 改动了时值前缀 | 将 `s1` (十六分) 写成 `q1` (八分) | 导致后续小节全部失步 |
| 删除了小节线 `\|` | 合并两个小节为一行 | 单行 8 拍 → 应 4 拍 × 2 |

> **最严重的问题**：LLM 在"纠错"过程中，常常把复杂节奏（混有 `d`/`s`/`q` 前缀的小节）整体替换为"看起来等价但节拍不同"的写法（例如上表第一行）。这会悄无声息地破坏拍数，从旋律上看仍然"像那么回事"，却完全无法编译。

---

## 提示词正文（复制给大模型使用）

````
你是一名简谱文本纠错助手。
下面提供的文件使用 jianpu-ly.py（https://github.com/ssb22/jianpu-ly）格式。
在纠错时，你必须严格遵守以下规则，否则生成的乐谱将完全无法编译。

==========================================================================
【一、文件结构（不可修改第1–4行）】
==========================================================================

文件的前 4 行（非注释行）固定格式为：

    % jianpu-ly.py      ← 第1行，固定标识，不得删除或更改
    title=歌曲标题       ← 第2行，标题声明
    1=C                 ← 第3行，调号（1=C / 1=G / 1=F / 1=Bb 等）
    4/4                 ← 第4行，拍号（4/4 / 3/4 / 6/8 等）

以 # 开头的行是注释，由工具自动处理，你不需要修改或删除注释行。

==========================================================================
【二、音符 token 格式】
==========================================================================

每个音符 token 由以下部分按顺序组成（均为 ASCII 字符）：

    [时值前缀] [升降号] 音名数字 [八度符号] [延音横线] [附点]

各字段说明：

  时值前缀（写在数字前面，用字母表示）：
    （无前缀）= 四分音符 = 1 拍
    q         = 八分音符 = 0.5 拍
    s         = 十六分音符 = 0.25 拍
    d         = 三十二分音符 = 0.125 拍
    h         = 六十四分音符 = 0.0625 拍

  升降号（写在数字前面）：
    #         = 升号（如 #1 = 升Do）
    b         = 降号（如 b3 = 降Mi）

  音名数字：1=Do  2=Re  3=Mi  4=Fa  5=Sol  6=La  7=Si  0=休止符

  八度符号（写在数字后面）：
    '         = 高八度（可叠加，如 1'' = 超高音）
    ,         = 低八度（可叠加，如 1,, = 超低音）

  延音横线（写在八度符号后面）：
    -         = 独立的 token（用空格分隔），每个 - 延长 1 拍

  附点：
    .         = 附点，使时值变为原来的 1.5 倍（如 q1. = 0.75 拍）

示例：
    1          四分音符 Do（1 拍）
    q1         八分音符 Do（0.5 拍）
    s1         十六分音符 Do（0.25 拍）
    1 -        二分音符 Do（2 拍，注意 - 是独立 token）
    1 - -      附点二分音符 Do（3 拍）
    1 - - -    全音符 Do（4 拍）
    q1.        附点八分音符 Do（0.75 拍）
    #1'        升号高八度 Do（四分音符，1 拍）
    qb3'       降号高八度 Mi（八分音符，0.5 拍）
    0          四分休止符（1 拍）
    q0         八分休止符（0.5 拍）

==========================================================================
【三、小节线规则】
==========================================================================

  用 | 分隔小节，它是独立的 token，两侧必须有空格：
      q1 q2 q3 q4 | q5 q6 q7 1 |

  每个小节内所有 token 的拍数之和，必须恰好等于拍号分子：
    4/4 拍 → 每小节恰好 4 拍
    3/4 拍 → 每小节恰好 3 拍
    6/8 拍 → 每小节恰好 6 × (0.125 拍) × 8 = 注意要用八分音符拍号

==========================================================================
【四、拍数自查表（4/4 拍 = 每小节 4.0 拍）】
==========================================================================

在提交修改前，你必须对每一个修改过的小节手动计算拍数总和：

  各 token 时值（拍）：
    q = 0.5   s = 0.25   d = 0.125   无前缀 = 1.0
    附点（.）= 原时值 × 1.5
    横线（-）= 作为独立 token 各加 1 拍

  拍数总和必须 = 拍号分子（4/4 → 4.0）

  示例验证：
    q1' qb2' qb3' qb3' qb2' q1' 0
    = 0.5+0.5+0.5+0.5+0.5+0.5+1.0 = 4.0 ✓

    q1' qb2' qb3' qb3' qb2' q1' 0 -     ← 错误示例（LLM 多加了 -）
    = 0.5+0.5+0.5+0.5+0.5+0.5+1.0+1.0 = 5.0 ✗

==========================================================================
【五、纠错操作白名单（只允许做以下修改）】
==========================================================================

✅ 允许修改：
  - 音名数字（如把 3 改成 2，因为 OMR 识别错了音高）
  - 升降号（如把 b3 改成 #2，因为等音替换）
  - 八度符号（如把 1' 改成 1，因为八度识别错了）
  - 注释文字（# 开头的行）

⛔ 绝对禁止（最常见的灾难性错误）：
  不得将某个小节的内容整体替换，即使你认为"等价"或"更工整"或"更简洁"。
  例如，你不得把：
      d1' d0 s0 sb2' s0 qb3' qb3' qb2' q1' s0 q0.     ← 原文（4.0拍，正确）
  改为：
      q1' qb2' qb3' qb3' qb2' q1' 0 -                  ← LLM 所谓的"简化"（5.0拍，错误！）
  哪怕原文看起来"节奏比较复杂"或"有多余的休止符"，也必须原样保留每个 token，仅修改音名数字。

❌ 严禁修改（会破坏拍数）：
  - 时值前缀（q/s/d/h）
  - 延音横线（- ）的个数
  - 附点（.）
  - 小节线（|）的位置或数量
  - 添加或删除任何音符 token（除非你同时删除等拍数的 token）
  - title= / 1=C / 4/4 等头部声明

==========================================================================
【六、输出格式要求】
==========================================================================

  - 输出整个文件，不要只输出修改部分。
  - 保持注释行（# 开头）原样不变。
  - 不要将多个小节合并到一行，也不要拆开原有的一行。
  - 每个 token 之间用单个空格分隔，不要用全角空格或制表符。
  - 不要在文件末尾添加额外的空行或说明文字。

==========================================================================
【七、修改后的自检步骤】
==========================================================================

在输出修改结果之前，请执行下列自检：

1. 找出拍号（第4行非注释行，如 4/4）。
2. 对每一个小节（两个 | 之间的内容），用如下方式计算拍数：
   a. 列出所有 token。
   b. 识别每个 token 的时值（含附点）。
   c. 对独立的 - token 各加 1 拍。
   d. 求和，确认等于拍号分子。
3. 如果某小节拍数不对：
   a. 找出新增或删除的 token 是否是拍数变化的原因。
   b. 撤销或修正，使拍数恢复正确。
4. 全部小节验证通过后，再输出修改结果。
````

## 在此文件中发现的具体错误（供参考）

以下错误由 LLM 纠错后引入，经 jianpu-ly.py 的 barcheck 报错确认。

| 位置 | 原（LLM 修改后）内容 | 实际拍数 | 问题 |
|---|---|---|---|
| 小节 1 | `q1' qb2' qb3' qb3' qb2' q1' 0 -` | **5.0 拍** | `0 -` 形成二分休止（2拍），共5拍 |
| 小节 3 | `q7 qb2' qb3'. sb3' q6 qb2' q1' 0` | **4.5 拍** | 多出一个八分音符时值 |
| 小节 5 | `q7 qb2' qb3' qb3' q#1' q1' 0 -` | **5.0 拍** | 同小节1，`0 -` 多出1拍 |

**jianpu-ly.py 实际报错**（由工具捕获）：
```
barcheck fail: note crosses barline at "1" with 0 beams
(16 skipped from 56 to 72, bypassing 64), scoreNo=1 barNo=4
```

**根因**：LLM 在修改音高时，将小节末尾的 `0`（四分休止，1拍）改写为 `0 -`（二分休止，2拍），且对小节3添加了额外的八分音符，均未做拍数验证。

## 使用方法

将上方 "提示词正文" 中代码块内的全部文字，粘贴到对 LLM 发起的请求**最前面**，随后附上需要纠错的文件内容即可。
