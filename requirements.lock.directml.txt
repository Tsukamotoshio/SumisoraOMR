# requirements.lock.directml.txt — DirectML 分发目标的锁定依赖清单
#
# 用途：面向"大部分 Intel/AMD 机器"的分发渠道。onnxruntime-directml 在任意
# DX12 GPU（Intel/AMD iGPU、AMD dGPU、NVIDIA）上走 DmlExecutionProvider，
# 无 GPU 时回退 CPU。打包 / 该分发渠道的新环境搭建用本文件。
#
#   pip install -r requirements.lock.directml.txt
#
# 生成方式（运行时已验证，从干净 lock 派生）：
#   从 requirements.lock.txt 机械派生，改动仅两处——
#     1. onnxruntime-gpu==1.26.0  →  onnxruntime-directml==1.24.4
#        （onnxruntime-directml 当前最高版本即 1.24.4；较 gpu 版 1.26 为降级）
#     2. 删除 3 个 CUDA EP 运行时包（DirectML 不需要）：
#        nvidia-cublas-cu12 / nvidia-cuda-nvrtc-cu12 / nvidia-cuda-runtime-cu12
#   其余传递依赖沿用 gpu 锁定的版本。
#
#   已验证（2026-07-23，Intel iGPU + onnxruntime-directml 1.24.4）：426 模型在
#   DmlExecutionProvider 与 CPUExecutionProvider 两路均成功推理，输出与 gpu-EP
#   CPU 金样**逐字节一致**；iGPU 较 CPU 约快 3×。onnxruntime-directml 的声明依赖
#   （flatbuffers/numpy/packaging/protobuf）与 gpu 版一致，故沿用的传递依赖版本
#   经该次运行确认可用。若日后升级任一运行时包，请在干净 directml venv 重新
#   freeze 覆盖本文件。（注：本机开发 venv 混入了 exp/jianpu-vlm-ocr 的
#   llama-cpp-python 及其 CUDA13 依赖，故不宜直接 freeze——须以本干净清单为准。）
#
# 与 requirements.lock.txt 的关系：那份含 onnxruntime-gpu，面向 NVIDIA/开发/CI。
