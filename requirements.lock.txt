# requirements.lock.txt — 完整锁定的依赖清单（构建可复现，P2-2）
#
# 来源：从经过完整回归验证的开发 venv 直接 `pip freeze` 生成（含传递依赖与
# 开发工具 ruff/pytest）。刻意不用 pip-compile 解析最新版：requirements.txt
# 的 >= 约束会解析到未经验证的新版本（实例：flet 0.85.3 曾导致 GUI 无法启动，
# 本项目适配的是 0.85.2）。锁定文件记录"已知可用"，而非"最新可装"。
#
# 用途：
#   打包 / CI / 新环境搭建： pip install -r requirements.lock.txt
#   日常表达意图仍看 requirements.txt（宽松约束）
#
# 再生成（升级依赖后）：
#   1. 在 venv 中升级目标包并跑完整回归（pytest + 实际转换一次）
#   2. .venv/Scripts/python -m pip freeze > requirements.lock.txt（保留本头部）
#
# 注意：本锁定含 onnxruntime-gpu（NVIDIA/开发/CI 目标）。DirectML 分发目标
# 需要另一份锁定（onnxruntime-directml 替换 onnxruntime-gpu），在准备该
# 分发渠道时按同样流程从对应验证环境生成。
#
# 已剔除交叉分支污染（发现于依赖漏洞扫描审计）：开发 venv 同时装过

# import 它），首次 freeze 把它和它的专属依赖一并冻了进来——包括无后缀的
# CUDA 13 运行时包（llama-cpp-python 的 cu132 wheel 专用；main 分支的
# onnxruntime-gpu 用的是 nvidia-*-cu12 这套，与之无关，予以保留）。
